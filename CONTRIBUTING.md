# 🤝 Contributing to TaskPilot

Thank you for your interest in contributing to TaskPilot! 🚀
We welcome contributions from developers of all skill levels.

---

## 📌 Getting Started

1. Fork the repository
2. Clone your fork:

   ```bash
   git clone https://github.com/YOUR_USERNAME/taskpilot-task-management.git
   ```
3. Navigate to the project:

   ```bash
   cd taskpilot-task-management
   ```
4. Install dependencies:

   ```bash
   npm install
   ```
5. Start the development server:

   ```bash
   npm start
   ```

---

## 🌿 Branching Strategy

* Create a new branch for every feature or bug fix:

  ```bash
  git checkout -b feature/your-feature-name
  ```

### Naming Convention:

* `feature/` → new features
* `fix/` → bug fixes
* `docs/` → documentation updates

---

## 🛠️ Code Guidelines

* Follow consistent code formatting
* Write clean, readable code
* Use meaningful variable names
* Keep components modular and reusable

---

## ✅ Commit Guidelines

Use clear and descriptive commit messages:

Examples:

* `feat: add drag and drop feature`
* `fix: resolve task deletion bug`
* `docs: update README`

---

## 🔁 Pull Request Process

1. Ensure your code works correctly
2. Run tests (if applicable)
3. Push your branch:

   ```bash
   git push origin feature/your-feature-name
   ```
4. Open a Pull Request

### PR Requirements:

* Clear description of changes
* Reference related issues (if any)
* Screenshots for UI changes (if applicable)

---

## 🐛 Reporting Issues

If you find a bug:

* Open an issue
* Provide steps to reproduce
* Include screenshots/logs if possible
